## What This Is

A single consolidated Obsidian plugin that replaces 4 separate plugins and adds Bloob Haus-specific features. Users can toggle each module on/off in settings. The goal is a smooth "notes workflow" for anyone using Bloob Haus — not yet publishing/sync (that's v2).

**v1 scope: notes workflow features only.** GitHub push / Magic Machine sync is deferred to v2.

## Module Detail

### Frontmatter (on by default)

Auto-populates YAML frontmatter when a new note is created.

**Fields injected on `vault.on('create')`:**
```yaml
---
date_created: 2026-05-01
date_updated: []
tags: []
bloob_object: note
---
```

- `date_created` — set once, never touched again
- `date_updated` — list; a new date is appended when a "significant" edit happens (see below)
- `tags` — empty array (user fills in)
- `bloob_object` — configurable default in plugin settings (starts as `"note"`)

**Significant edit detection** (from File Timestamp Tracker, `file-timestamp-tracker/`):
- Listen for `vault.on('modify')`
- Track character count delta since last check
- If delta ≥ threshold (default: 20 chars) → append today's date to `date_updated`
- Once per day max — if today is already in the list, skip
- Never add to files that existed before the plugin was installed (check if `date_created` is absent → skip `date_updated` tracking)

**Excluded folders** (configurable, default: `_media`, `templates`):
- Skip frontmatter injection for files in these folders

**Settings:**
- `bloob_object` default (text input, default `"note"`)
- Significant change threshold (number, default `20`)
- Excluded folders (comma-separated list)

---

### Image Zoom (on by default)

Click an image in reading view to open a fullscreen overlay. Scroll to zoom. Drag to pan.

- Ported directly from `image-zoom/`
- Skips emoji images
- Escape or click overlay to close
- 5px minimum drag distance before treating as drag (not accidental close)

No user-configurable settings needed for v1.

---

### Date Keywords (off by default)

Type a trigger word and it gets replaced with a formatted date/time.

**Default triggers** (all configurable):

| Keyword | Replacement |
|---------|-------------|
| `TODAY` | `YYYY-MM-DD` |
| `DATE` | `MMMM D, YYYY` |
| `TIME` | `HH:mm` |
| `WDATE` | `dddd, MMMM D` |
| `TDATE` | `[Today is] MMMM D` |
| `TTDATE` | `[Today is] dddd, MMMM D, YYYY` |

- Ported from `today-replacer/`
- Also auto-renames files if the first line contains a trigger word
- Settings: add/remove/edit trigger → format pairs; includes format cheat sheet

---

### Link Encoder (off by default)

Paste a file path or URL → automatically converts to a formatted markdown link.

- File paths → `[📄 filename](path%20with%20spaces)`
- Folders → `[📁 foldername](path)`
- Web URLs → `[🌐 url](url)`
- Handles Windows paths, UNC paths, `file://`, HTTP/HTTPS
- Command: "Update all file/folder links in this note"
- Ported from `obsidian-link-encoder/`

No settings for v1 (the icon prefix style could be configurable later).

## Feature Requests & Bug Reports

A "Feedback" button in the plugin settings tab.

**UX:**
1. User clicks "Submit feedback or bug report"
2. A small modal opens with two fields: Subject (short text) + Description (textarea) + a "Bug" / "Feature request" toggle
3. On submit: opens a pre-filled GitHub Issues URL in the browser

```
https://github.com/LSanten/obsidian-bloob-haus/issues/new
  ?template=feedback.md
  &title=[BUG]%20{subject}   (or [FEATURE])
  &body={description}
```

No API key, no infrastructure. User sees their submission on GitHub, can track it. This is the approach used by many community plugins.

**GitHub issue templates to create:**
- `bug_report.md`
- `feature_request.md`

## Plugin Architecture

```
obsidian-bloob-haus/
├── manifest.json
├── main.ts                  # Plugin entry — loads enabled modules
├── settings.ts              # Settings schema + unified settings tab
├── styles.css               # All CSS (image zoom overlay, etc.)
├── modules/
│   ├── frontmatter.ts       # Auto-frontmatter + date_updated tracking
│   ├── image-zoom.ts        # Zoom/pan overlay
│   ├── date-keywords.ts     # TODAY/DATE replacer
│   └── link-encoder.ts      # Paste-path → markdown link
└── ui/
    └── feedback-modal.ts    # Feedback/bug report modal
```

**Settings schema:**
```typescript
interface BloobHausSettings {
  modules: {
    frontmatter: boolean;      // default: true
    imageZoom: boolean;        // default: true
    dateKeywords: boolean;     // default: false
    linkEncoder: boolean;      // default: false
  };
  frontmatter: {
    bloobObjectDefault: string;          // default: "note"
    significantChangeThreshold: number;  // default: 20
    excludedFolders: string[];           // default: ["_media", "templates"]
  };
  dateKeywords: {
    triggers: { keyword: string; format: string }[];
  };
}
```

Each module class receives the settings object and the Obsidian `app`. Modules register their own event listeners and clean up in `onunload()`.

## How Bloob Haus URLs Are Constructed (for future "copy link" / publish features)

When a future plugin module needs to generate a public URL for the currently open note, here's how the URL is determined:

### URL components

```
https://{site.url}/{vault-relative-path-without-ext}/
```

**`site.url`** — read from `sites/{site-name}.yaml` in the webapp repo:
```yaml
site:
  url: "https://leons.bloob.haus"
```

**Vault-relative path** — Obsidian's `file.path` gives this directly (e.g. `marbles/CHOMSKY-GLOBAL-HEG-VID.md`). Strip `.md`, replace spaces with hyphens per segment, join with `/`.

### Key facts confirmed from codebase (2026-05-18)

- **Case is preserved.** `CHOMSKY-GLOBAL-HEG-VID.md` → URL segment `CHOMSKY-GLOBAL-HEG-VID`. Do NOT lowercase.
- **Spaces → hyphens.** `preprocess-content.js` sanitizes filenames when writing to `src/` (`seg.replace(/ /g, '-')`). Eleventy then generates clean URLs. Match this in any plugin that constructs URLs.
- **No other transformation.** Don't slugify, don't `encodeURIComponent` — just the spaces→hyphens replacement.
- **Folder structure maps directly to URL path.** `say-hello-to/foo.md` → `/say-hello-to/foo/`. No flattening.
- **`slugify` in the codebase is for link resolution only** (wiki-link lookup, not URL generation). Don't use it for constructing public URLs.

### Where to find the site URL in future plugin versions

The plugin will need to know `site.url` to build links. Options in priority order:

1. **Plugin setting (v1 approach):** User pastes their site URL into the plugin settings once (e.g. `https://leons.bloob.haus`). Simplest and works offline.
2. **Read from `_bloob-settings.md` in the vault** (v2): The vault's `_bloob-settings.md` could include a `site_url` field — the plugin reads it on load. Keeps config in one place.
3. **Fetch from webapp `sites/*.yaml` via GitHub API** (v2+): More automatic but requires auth and a network call.

For v1 (copy-link-button and future "Copy public URL" command), use option 1: a single settings field `bloobHausSiteUrl` defaulting to empty, with a placeholder hint like `https://yourdomain.com`.
